爬取流浪地球6万条影评

发表于 2019-02-18 | 分类于 CS学习日志 | | 阅读次数:

此次爬虫背景

2019年2月5日上映的3D电影《流浪地球》在国内上映。截至2月17日，《流浪地球》票房已达到36.51亿，超过《红海行动》的36.50亿，成为目前内地影史票房亚军，正在向内地影史票房冠军《战狼2》的56.8亿目标冲刺。

我想做什么

关于《流浪地球》，网友褒贬不一，有人大声叫好，有人大骂吴京……
我想知道这个电影到底怎么样，所以决定用python爬取猫眼电影近7天的6万条影评数据，来看看大家都在说什么。

毕竟，实践出真知！哈哈哈哈哈

先来展示成果(数据可视化)

以下图表均由61606条影评分析得到



如上图，给10分的人数最多，很少有人给8分一下



如上图，流浪地球的平均得分是9.217分



如上图，这6万条影评发布者，主要是等级为2、3、4的猫眼用户

如上图，不同等级的用户所给的分数也不同



如上图，38%为男性观众、37%为女性观众、剩下25%的观众未设置性别



如上图，是这6万条影评中被提及到的词语（被引用次数越多，字越大）



如上图，被引用次数最多的10个词（好看、中国、不错、好、特效、科幻片、震撼、国产、科幻、非常），其中“好看”被引用19042次，被大家普遍提及。


第一次尝试（爬取豆瓣影评）

篇幅有限，这里只是简单记录一下思路吧！



首先，登陆网页版猫眼电影，F12开发者模式中选择移动端模式（箭头所指），因为网页版只能显示10条影评。影评有很多页，每次翻页后，XHR中就会出现一条JSON数据（如上图），offset代表偏移量，以此时此刻为基准点，每翻一页，offset值就会加15，所以可以利用此规律模拟cookie登陆网站爬取JSON数据。



如上图，每条JSON数据包中，就可以得到我们想要的数据（评论者的昵称、性别、账号ID、评论内容、评论时间等）



起初，爬取豆瓣影评，发现豆瓣只给500条影评……（500条数据，能干什么啊）




如上图，通过jieba、pandas等第三方包分析出这500条影评中，出现次数最多的词语

第二次尝试（爬取猫眼影评）

第二次，我选择爬取猫眼电影近7天的影评，并得到6万条影评数据



如上图，爬取2月10日——2月17日的数据（耗时1+小时）



利用MongoDB采集数据（共61606条影评）



将数据导出为CSV，用excel2019打开，利用excel将数据可视化



评论内容单独导出，准备用python制作词云



可以看到，被引用次数最多的10个词（好看、中国、不错、好、特效、科幻片、震撼、国产、科幻、非常），其中“好看”被引用19042次，被大家普遍提及。

总结

第一次爬取这么多数据，耗时2天，不断地遇到新的问题。爬取网页时，因为短时间内频繁访问猫眼网站，被网站拒绝访问过；二次安装MongoDB运行环境时，总是无法关闭MongoDB Service；数据存到MongoDB中时，突然出现【TimeOutError10061】连接不到主机；准备利用阿里云DataV进行数据可视化，然后发现提交的CSV文件不能超过512KB（我的CSV有6MB）……总之，我的想法算是实现啦，哈哈哈哈哈哈哈哈，开心！
